1. INTRODUCTION
The goal of this engagement is to find possible existing vulnerabilities that are applicable to Web Application and verify if they are exploitable, evaluate its risk in correspondence to its adverse impact on the business organization, provided that a permission to exploit the vulnerability is granted by the client.
  1. Strobes security team will inspect the web application and map its functionality.
  2. The team will develop a thorough test plan with test cases and their scenarios to execute against the target web application after mapping out the functionalities and use cases of the application.
  3. The security team will assess the web application defined in the scope for the OWASP Top 10 vulnerabilities & other application security issues using OWASP Web Application Testing Methodology.
  4. The security team will report the progress of the assessment to the {{ info.organization_name }} periodically.
  5. If a vulnerability is found, the security team will analyze the Exploitability and Impact Metrics corresponding to the Common Vulnerability Scoring System (CVSS) Framework and then assign the severity.
  6. The security team will produce an audit report that includes the overall security posture of the targeted web application, a description of the vulnerabilities, recommendations on how to address the issues found during the assessment, and overall risks that are associated due to them.
  7. Once a re-validation request for the reported vulnerabilities is raised, the security team will revalidate the reported issues.
  8. The security team will produce a follow-up audit report, which will reflect the status of the reported issues after the mitigations are deployed by the {{ info.organization_name }}. Strobes security team will ensure that all the mitigations deployed in the web application are thoroughly analyzed and possible methods to bypass the mitigations are being checked.
  9. All information obtained during the assessment will be processed, analyzed, and stored in accordance with the Strobes security practices and data handling policy.

GLOSSARY
CLOUD SECURITY CONFIGURATION REVIEW: Cloud security configuration review involves the assessment of the security controls and settings implemented in a cloud environment to identify any vulnerabilities or risks. The review involves assessing access controls, network security, encryption, data backup, and other security controls to ensure they are effective and in compliance with relevant regulatory and compliance standards.
VULNERABILITY ASSESSMENT: Vulnerability assessment is the process of identifying and evaluating the security weaknesses and vulnerabilities in a system, network, application, or other technology infrastructure. The goal of vulnerability assessment is to identify and prioritize vulnerabilities, assess the exploitability and impact of a successful attack, and provide recommendations for remediation to reduce the overall risk to the system.
PENETRATION TESTING: Form of assessment to identify ways of exploiting vulnerabilities to circumvent or defeat the security features of system components. Penetration testing includes network and application testing as well as controls and processes around the networks and applications and occurs from both inside and outside the environment (internal/external testing).
BLACK BOX PENETRATION TESTING: Type of penetration testing in which an assessor evaluates security controls by simulating a real attack targeting an application. Black Box techniques assess the security of individual high-risk compiled components; interactions between components and interactions between the entire application or application system with its users, other systems and the external environment.
INTERNAL PENETRATION TESTING: Type of penetration testing that simulates what an insider attack could accomplish. The target is typically the same as external pen-testing, but the major differentiator is the "attacker" either has some sort of authorized access or is starting from a point within the internal network. Insider attacks have the potential of being much more devastating than an external attack because insiders already have the knowledge of what's important within a network and where it's located, something that external attackers don't usually know from the start
